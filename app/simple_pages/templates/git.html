Git and GitHub

GitHub is an online website that assist programmers in developing software and websites by helping to collaborate code and tracking any changes made by all parties involved. Some of the essential aspects of GitHub include repositories, commits, branches, pushes, and pull requests.

  • Repositories - A repository contains all your files and tracks any changes..
  • Commit - A commit is a revision or change to a file or files. When a commit is made, a unique ID is created to track who made the change and when.
  • Branch - A branch is a parallel version of a repository. It allows the programmer to make and test any changes to a website without disrupting the live site. When the programmer is satisfied with their changes, the branch can be merged into the main branch.
  • Pushes - A push is when you commit any local changes to the remote repository.
  • Pull - When you pull from GitHub, you are importing files from a remote branch into your local branch. This is especially important when there are other collaborators to ensure your software if up to date with the latest changes.
  • Tags - Tags are associated with commits, so you can use a tag to mark an individual point in your repository's history, including a version number for a release.
