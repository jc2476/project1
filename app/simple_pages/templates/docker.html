Docker

Docker is a platform that allows developers to package and run an application in an environment called a container. The docker container environment can be used for testing an application or for deployment of the final product to run either locally or in the cloud. In the past, we applications were run on physical servers. If your site became popular, you may have had to purchase additional servers to balance the load of the incoming traffic. Today, there is no longer a need to purchase any hardware since the inception of “the cloud.” Cloud hosting is comprised of an array of computers that manage and balance resources through software called containers. To better understand Docker, there are several terms that are commonly used and should be understood:

  • DockerFile – A simple text file that contains instructions on how to build the container image.
  • Docker images – The image contains the source code as well as the tools and libraries that the application needs to run as a container.
  • Containers – The container is a live running instance of an image. Users can interact with containers and administrators can adjust settings and conditions.
  • Docker Hub – Docker Hub is the term used for the public repository of container images.
  • Docker Compose – A command used to manage the application’s architecture. It creates a YAML file, which specifies the services that are in an application.

    • The Docker Hub repository for this project can be found at https://hub.docker.com/r/jcasorio/project1-prod.

    Docker Image

    An image is a reach only template with instruction to create a docker container. To create an image, you need to create a Dockerfile with the instructions that outlines the steps needed to create the image and run it.


    Docker Container

    A container is a runnable instance of an image. A container is isolated from other containers and the host machine. They are small and lightweight environments that share the operating system kernel and resources.


    Common Docker commands:

  • docker run -itp "port #" "repository name:tag" - This command creates a container from the image
  • docker compose up --build" - Create and start all the services in the configuration
  • docker stop "container id" - This stops a running container
  • docker kill "container id" - This stops a running container immediately
  • docker ps - Lists all the running containers
  • docker images - Lists all the docker images
  • docker pull "image name" - This will pull an image from the docker repository to the local machine
  • docker exec -it "container name" [user command] - Runs a command specified by the user in a container
