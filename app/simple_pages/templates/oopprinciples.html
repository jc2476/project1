OOP Principles

Glossary


Terms

Instantiation - The creation of a real instance or particular realization of an abstraction or template such as a class of objects or a computer process.
Object - Things you think about first in designing a program and they are also the units of code that are eventually derived from the process.
Class - A template definition of the methods and variables in a particular kind of object.
Namespace - A declarative region that provides a scope to the identifiers (the names of types, functions, variables, etc) inside it.
Constructor - A special method of a class or structure in object-oriented programming that initializes a newly created object of that type.
Fixture - Functions which will run before each test function to which it is applied.
Type Hint - A formal solution to statically indicate the type of a value within your Python code.
Type Cast - Changing an variable of one data type into another.
Unit Test - A software testing method by which individual units of source code—sets of one or more computer program modules together with associated control data, usage procedures, and operating procedures—are tested to determine whether they are fit for use.
Static - A keyword that can be used to declare a member of a type so that it is specific to that type.
Instance Method - Methods which require an object of its class to be created before it can be called.
Instance Property - A typical property declared in a class.
Static Method - A method defined as a member of an object but is accessible directly from an API object's constructor, rather than from an object instance created via the constructor.
Static Property - The properties inside a static method shared by all instances of a class.
Encapsulation - The bundling of data with the methods that operate on that data, or the restricting of direct access to some of an object's components.
Inheritance - The mechanism of basing an object or class upon another object or class, retaining similar implementation.
Polymorphism - The provision of a single interface to entities of different types, or the use of a single symbol to represent multiple different types.
Abstraction - Used to hide background details or any unnecessary implementation about the data so that users only see the required information.

Overview of Design Patterns

Design matterns are typical solutions to common problems in software design. The can be thought of as pre-made blueprints that can be customized to fit a users code. The pattern is a general concept for solving a particular problem and can't just be copied into your program. Design patterns can be thought of as a toolkit with tried and tested solutions to common problems in software design. Patterns are classified into three main groups:

Creational patterns - Provide object creation mechanisms that increase flexibiliyt and reuse of existing code.
Structural patters - These explain how to assemble objects and classes into larger structures, while keeping the structures flexible and efficient.
Behavioral patterns - Behavioral patterns take care of effective communication and the assignment of responsibilities between objects.
